**Narration Objective:**
- Describe movement versus previous averages by calling out the strongest increases, decreases, and stable patterns within the provided buckets.

**Extraction Steps:**
1. Read currentScheduleId and prevScheduleCount.
2. For each bucket, use the provided list as already sorted by rule.
3. Extract up to 3 items from each bucket; if a bucket is empty, state "Not Available" for that bucket.
4. For each item, extract subAuditName, currentIssueCount, previousAverageIssueCount, changeCount, changePercent, direction.

**Required Data Points:**
- currentScheduleId, prevScheduleCount
- For each bucket item: subAuditName, currentIssueCount, previousAverageIssueCount, changePercent

**Coverage Rule:**
- Do not cross buckets. Use only items in their respective lists.
- If prevScheduleCount is 0, state that previous schedule comparison is "Not Available" and do not mention changePercent values.

**Must Highlight:**
- At least one item from each non-empty bucket.
- The largest increase (from increasing) and largest decrease (from decreasing) if those buckets are non-empty.
- Clearly indicate when a bucket is empty using "Not Available".

**Comparison Logic:**
- Increasing bucket: report largest increases as provided.
- Decreasing bucket: report largest decreases as provided.
- Stable bucket: report smallest absolute changes as provided.

**Output Format:**
- **Trend Movement:** 30–60 words.
